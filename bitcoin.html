How Bitcoin Works

You probably already know what Bitcoin is. In this post, you will also learn how it works.

Why Trust Matters

Trust is important when it comes to money, but not as important in other domains. Consider another application such as messaging. If I am sending a message to my friend, it would kinda suck if a bad actor interfered with my message. It usually wouldn't be the end of the world, but it would be annoying.

Also, what incentive would someone have to interfere with messages between me and my friend? In most cases, there would be little incentive. It would be bizarre for someone to go out of their way to hack my messages.

Money is a highly adversarial domain. There is a large incentive to hack money. If you successfully hack someone's money, you earn money, so we need to trust that our money is safe.

How Trust is Instilled

In the case of U.S. Dollars, we trust that dollars are safe because the U.S. government uses violence to enforce the rules. If you steal money, you go to jail.

In the case of Bitcoin, we trust Bitcoin for two reasons:

  • Cryptography
  • Transparency

I will expand on what these mean below.

Cryptography 101

I will explain the minimum of what you need to understand this post. I will also leave some parts out, because cryptography is a very rich field of mathematics.

Public and Private Keys

In cryptography, there are these things called public keys and private keys. You choose a random private key, and then keep it a secret. A private key is kind of like a password.

With your private key, you can use a cryptographic function to generate a public key. The key generation function is one-way, which means that it is easy to go from private key to public key, but it is hard to go from public key to private key.

Theoretically, it would be possible to derive the private key from the public key, but it would take so long that it is not practical. Therefore, we say that it is practically impossible.

Another special property of public and private keys is the notion of signatures. You can sign some data with your private key, and then anyone can use your public key to verify that the data was signed by you. And that can be done while keeping your private key secret. That is why one of the keys is public and the other is private.

Hashing

Hashing is a cryptographic function that takes some data and produces something called a hash. The hash function is one-way, which means that it is easy to go from data to hash, but it is hard to go from hash to data. Similar to the key generation function, reversing the hash function is theoretically possible, but practically impossible.

Nonces

A nonce is a random number. It is used in cryptography to make it hard to guess the input to a cryptographic function. For example, if you want to sign some data, you can use a nonce to make it hard to guess the private key.

What does Bitcoin guarantee?

  1. I am the only person who can spend my Bitcoin. In other words, nobody can steal my Bitcoin
  2. The same Bitcoin can only be spent once. The notion of spending the same money twice is called the double-spend problem

How are transactions secured?

The first guarantee is that I am the only person who can spend my Bitcoin." How is this enforced?

We use public and private keys to ensure that I am the only person who can spend my Bitcoin. When I want to spend my Bitcoin, I sign a transaction with my private key. Then, anyone can use my public key to verify that the transaction was signed by me. If someone tries to fraudulently claim that I sent some Bitcoin, others can verify with my public key and see that it's a lie.

What about the double-spend problem?

The second guarantee is that any particular Bitcoin can only be spent once. How is this enforced?

Let's say I receive some Bitcoin. When I send that Bitcoin to someone else, that transaction is recorded on the Bitcoin blockchain. The blockchain is a public ledger of all transactions. Then if I ever try to send that same Bitcoin, others will see that I already spent it.

What is the blockchain?

A blockchain is a chain of blocks. Each block contains a list of transactions. The blocks are chained together in a way that makes it practically impossible to modify previous transactions. In fancy terms, we can describe the blockchain as immutable.

It is very important for the blockchain to be immutable. If it was possible to go back and modify previous transactions, then I would be able to spend the same Bitcoin twice.

How are new blocks added to the chain?

Now you know that the blockchain is a chain of blocks, and the old blocks cannot be changed. You might be wondering how new blocks are added to the chain.

There are these things called miners. They are people or businesses who run their computers to compete to add new blocks.

It costs money to run computers, so now you are probably wondering why people are willing to compete to add new blocks Whenever someone wins the competition, they earn some Bitcoin. So all these miners are mining as fast as possible so they can earn a Bitcoin reward as compensation for their mining.

Now you might be wondering what it looks like for a miner to win. The miner who wins is the one who finds a nonce that satisfies certain conditions. Miners are trying to find a nonce that results in a hash that starts with a certain number of zeros. The number of zeros is called the difficulty. The difficulty is adjusted occasionally such that blocks are produced roughly every 10 minutes. If blocks are being produced quicker than 10 minutes, for example, then the difficulty is increased, and the hash is required to start with more zeros.

In order to find this nonce, miners use a brute force approach. They try a random nonce, verify whether the hash starts with the correct number of zeros, and if not, they try another random nonce.

Why is the blockchain immutable?

The blockchain is structured such that the hash of the previous block is passed into the hash function for the current block. So let's say I try to modify a block from 10 blocks ago, and I claim that is the correct chain. Others can easily recompute the hashes and see that I am lying.

Now let's say I am a little more savvy. To convince others that my version is the correct chain, I will find valid nonces for all 10 blocks. Through this process, it is theoretically possible to modify the chain. But for practical purposes, it is expensive to modify the chain, and after enough blocks have passed it is practically impossible.

Once a new block is added to the chain, it is in the miners best interest to try to add new blocks to the longest known version of the blockchain. If they win and add new blocks, they earn the Bitcoin reward. If a miner tries and fails to modify the chain, they are only hurting themselves by wasting time and money.
